# Folder strcuture of the frontend application:

## Assets
all the static assets like images, icons, fonts, etc. are stored in the `public` directory. This allows for easy access and management of these resources.

## Components
The `components` directory contains reusable React components that can be used across different pages of the application. This promotes code reusability and helps maintain a consistent design throughout the app.  

## Hooks
The `hooks` directory contains custom React hooks that encapsulate reusable logic. This allows for cleaner and more modular code, as well as easier testing and maintenance.

## Styles
The `styles` directory contains all the CSS files for the application. This includes global styles, as well as component-specific styles. Organizing styles in this way helps keep the codebase clean and

makes it easier to manage and update the styling of the application.

## Utils
The `utils` directory contains utility functions that can be used throughout the application. These functions can include things like API calls, data formatting, or any other helper functions that are needed in multiple places in the app. This promotes code reusability and helps keep the codebase organized.

# State Management
